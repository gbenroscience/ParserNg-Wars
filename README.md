
# ParserNG-Wars
This repository is designed to showcase the battles, or if you will, the performance of ParserNG against other Java math parsers.

## NOTE if you are an avid user of ParserNG, the version used in this test is from the future as of today(March 3rd 2026)
By futuristic, we mean that it is not yet available on maven central. 
The latest version on maven cetral is v0.2.3, but the one used in this test is version 0.2.4. On the github repo, this futuristic version is available on the token-evaluator branch.

This version is easily 10x faster than the version on master, but we will push it to maven central once all tests have been run on it. It has passed our suite of tests

The libs folder contains the executable jars of all math libs that will perform in the benchmarks, and the code for the shootout is in
the `Benchmark.java` class.
You may run the project by running the Benchmark.java class in your IDE or building the project as an executable jar in your IDE and then doing:
java -jar ParserNG-Wars.jar

### Typical input:

```Java
   String s11 = "(sin(3)+cos(4-sin(2)))^(-2)";

        new Benchmark().runBenchmarks(s11, new int[]{1, 5, 10, 50, 100, 500, 1000, 5000, 10000, 20000, 50000, 100000, 200000, 500000, 1000000, 2000000, 5000000, 10000000},
                PARSER_NG, JAVA_MEP, EXP_4J);
```
